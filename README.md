# FitPro (v1.0.1 - 2026-03-20)

안티그래비티 팀 여러분, 현재 프로젝트의 맥락을 빠르게 파악하고 즉시 개발에 활용할 수 있도록 정리했습니다.

### 1. 프로젝트 요약
- **목적**: '마라톤 완주'를 목표로 하는 사용자들을 위한 개인 최적화 건강·운동 가이드 레이어.
- **핵심 가치**: OCR을 통한 러닝 기록 자동 입력, 캘린더 연동을 통한 일정 최적화, 정밀한 신체 데이터 변화 예측.
- **타겟 장비**: 삼성 Galaxy S26 (390x844 뷰포트) 최적화.

### 2. 기술 스택
- **프론트엔드**: HTML5 (Semantic), Vanilla CSS (Custom Variable 기반), Modern JS (ES6+).
- **백엔드**: Node.js + Express.
- **데이터베이스**: MariaDB (Docker 기반).
- **인프라**: Cloudflare Pages (프론트), 로컬 Docker (백엔드/DB), SSH 터널링 (모바일 테스트).

### 3. 디렉터리 주요 구조
```bash
fitpro-app/
├── index.html           # 메인 UI 구조 및 모달 레이아웃
├── app.js               # [핵심] 상태 관리, API 연동, 날짜/라벨 업데이트 로직
├── style.css            # [디자인] 글래스모피즘, 모바일 반응형(매체 쿼리) 디자인
├── server/
│   ├── server.js        # Express API 엔드포인트 및 게스트 모드 인증 우회
│   └── db.js            # MariaDB 커넥션 풀 설정
└── .agents/workflows/   # 안티그래비티 전용 태스크 워크플로우
```

### 4. 로컬 개발 시작하기
1. **DB 가동**: `docker-compose up -d` (MariaDB 컨테이너 가동)
2. **백엔드**: `cd server && npm install && npm start` (5000포트)
3. **프론트엔드**: `index.html`을 라이브 서버로 실행 (8080 또는 3000포트)
4. **모바일 테스트**: `app.js` 최상단 `API_BASE` 로직 확인 (자동으로 로컬/프록시 주소 결정)

### 5. 현재 개발자 필독 사항 (API 우선순위)
- 앱은 다음 순서로 API 주소를 결정합니다:
  1. URL 쿼리 파라미터 `?set_api=...`
  2. 로컬스토리지 `fitpro_api_url`
  3. 현재 브라우저의 접속 주소 + `/api` (기본값)
- **모바일 로그인**: 현재 `bypass_token`을 통해 로그인 없이 게스트 모드 진입이 완벽히 구현되어 있습니다.
